## Part 2. ExpenseOne 개발에 도움이 되는 도구 평가

### 즉시 도입 추천 (HIGH)

| 도구 | ExpenseOne에서의 활용 |
|-----|---------------------|
| **Lazygit** | GitHub 연동 완료된 상태에서 커밋 히스토리 시각 관리, 브랜치 전략 수립 시 필수. 현재 auto-commit hook이 설정되어 있어 커밋 이력이 빠르게 쌓일 예정 — Lazygit으로 이력 관리가 훨씬 편해짐 |
| **Glow** | CLAUDE.md, PRD.md, REVIEW 보고서 등 마크다운 문서를 터미널에서 바로 확인. Claude Code 작업 중 문서 참조 시 유용 |
| **Zoxide** | `expense-app/src/components/forms` 같은 깊은 경로를 `z forms`로 한번에 이동. 프로젝트 구조가 깊어 체감 효과 큼 |

### 선택적 도입 (MEDIUM)

| 도구 | ExpenseOne에서의 활용 |
|-----|---------------------|
| **Btop** | Vercel dev 서버 + Supabase 로컬 실행 시 리소스 모니터링. 성능 디버깅 시 유용 |

| **CSV Lens** | 경비 데이터 CSV 내보내기 기능 추가 시, 터미널에서 바로 데이터 검증 가능 |

|-----|------|

## Part 4. ExpenseOne UX 핵심 리뷰 — "채울 칸이 너무 많다"

### 현재 폼 필드 수

| 폼 | 필수 | 선택 | 총계 |
|----|------|------|------|
| 법카사용 | 3개 | 5개 | **8개** |
| 입금요청 | 7개 | 6개 | **13개** |

### 현재 입력 플로우

```
[유형 선택] → [폼 작성 (단일 페이지)] → [제출 완료]
     1단계           2단계                  3단계
```

**법카사용 필수 입력**: 제목 → 금액 → 카테고리 (3개)
**입금요청 필수 입력**: 제목 → 금액 → 카테고리 → 은행 → 예금주 → 계좌번호 → 파일첨부+문서유형 (7개+)

### Slack 채널 글쓰기 vs ExpenseOne 폼 비교

| 항목 | Slack 채널 글쓰기 | ExpenseOne 폼 |
|------|-----------------|---------------|
| **입력 시간** | ~30초 (자유 텍스트) | ~2~3분 (구조화된 폼) |
| **입력 부담** | 낮음 (한 줄이면 충분) | 높음 (13개 필드) |
| **데이터 정확성** | 낮음 (오타, 누락, 형식 불일치) | 높음 (Zod 검증, 필수값 보장) |
| **검색/조회** | 어려움 (Slack 검색 한계) | 쉬움 (필터, 정렬, 대시보드) |
| **승인 프로세스** | 없음 (관리자가 읽고 수동 처리) | 자동 (알림 → 원클릭 승인) |
| **통계/분석** | 불가능 | 자동 (월별, 카테고리별, 팀별) |
| **영수증 관리** | Slack에 파일 흩어짐 | 건별 첨부, 영구 보관 |
| **모바일 경험** | Slack 앱에서 편리 | PWA이지만 폼 입력은 번거로움 |
| **이력 추적** | 스크롤로 찾기 | 상태별 필터 (제출/승인/반려) |

### 솔직한 평가

**Slack이 더 편한 경우:**
- 법카사용 단순 보고 ("점심 15,000원")
- 빈도가 낮을 때 (월 1~2건)
- 팀원이 5명 이하일 때

**ExpenseOne이 압도적인 경우:**
- 입금요청 (계좌정보, 증빙서류 필수)
- 월말 정산/리포트 생성
- 10명 이상 팀원의 경비 추적
- 감사/회계 증적 필요 시

### 결론: "현재 ExpenseOne은 Slack보다 편한가?"

> **법카사용: 솔직히 Slack이 더 빠르다** (필수 3개 필드지만 앱을 열고 → 유형 선택 → 폼 작성하는 과정 자체가 Slack 한 줄보다 무겁다)
>
> **입금요청: ExpenseOne이 확실히 낫다** (계좌정보/증빙 관리를 Slack으로 하면 오히려 혼란)

## Part 5. "채울 칸이 많다" 문제 해결 방안

### 방안 1: Slack → ExpenseOne 자동 입력 (가장 임팩트 큼)

```
크루가 Slack에 자연어로 작성:
"점심 교보문고앞 김밥천국 15000원"

→ Slack Bot이 파싱:
  제목: "점심 교보문고앞 김밥천국"
  금액: 15,000원
  카테고리: 기타 (자동 추정)

→ ExpenseOne에 자동 등록 (법카사용)
→ Slack에 확인 메시지: "✅ 등록 완료 [수정하기]"
```

**장점**: 기존 Slack 습관 유지 + 데이터는 ExpenseOne에 구조화
**구현 난이도**: 중 (Slack Events API + 자연어 파싱)

### 방안 2: 법카사용 "빠른 입력" 모드

현재 8개 필드 → **3개로 축소**한 간소화 모드:

```
┌─────────────────────────┐
│  💳 법카사용 빠른 입력    │
│                         │
│  뭐 샀어?  [점심 김밥천국] │
│  얼마?    [15,000원]     │
│  📷 영수증 (선택)        │
│                         │
│  [등록]                  │
└─────────────────────────┘
```

- 카테고리: "기타"로 기본값 (나중에 수정 가능)
- 가맹점명/설명: 제목에서 자동 추론 또는 생략
- VAT/원천징수: 기본 미체크 (나중에 수정 가능)

**장점**: 모바일에서 10초 안에 등록 가능
**구현 난이도**: 하

### 방안 3: 영수증 사진 → AI 자동 입력

```
📷 사진 촬영 → AI OCR 분석 →
  가맹점: "김밥천국 종로점"
  금액: 15,000원
  날짜: 2026.03.22
  → 폼 자동 채움 → 크루는 확인만
```

**장점**: 입력 필드 0개 (사진만 찍으면 끝)
**구현 난이도**: 중~고 (Claude Vision API 또는 Tesseract OCR)

### 방안 4: 입금요청 계좌정보 자동완성 강화

현재도 "최근 계좌 5개" 저장 기능이 있지만:
- **자주 쓰는 계좌 즐겨찾기** → 원터치 선택
- **수취인 별명** → "디자이너 홍길동" 선택하면 은행/계좌/예금주 자동 입력
- **팀 공용 수취인 목록** → 관리자가 등록한 거래처 목록에서 선택

### 방안 5: 반복 경비 템플릿

```
[월 정기] 사무실 청소비 → 매월 1일 자동 생성 (크루는 확인만)
[주 정기] 간식비 → 매주 금요일 자동 생성
```

## Part 7. 최종 결론

### 영상의 CLI 도구들이 ExpenseOne에 주는 가치

영상의 10개 도구 중 **Lazygit, Zoxide, Glow 3개**가 즉시 가치를 제공합니다.
나머지는 AI 모델 관리/시스템 모니터링 도구로 ExpenseOne 개발에는 간접적 도움입니다.

### "채울 칸이 많다" 문제에 대한 답

**현재 상태**: 입금요청은 ExpenseOne이 확실히 우위. 법카사용은 Slack이 더 빠름.

**해결 방향**: Slack을 대체하는 것이 아니라 **Slack과 공존**하는 전략.
- 법카사용은 Slack에서 자연어로 → ExpenseOne이 자동 수집
- 입금요청은 ExpenseOne 폼 유지 (구조화된 데이터 필수)
- "빠른 입력" 모드로 법카사용의 진입 장벽을 Slack 수준으로 낮추기

> **핵심**: 도구의 가치는 "입력의 편리함"과 "데이터의 정확함" 사이의 균형에 있다.
> ExpenseOne은 후자에서 압도적이지만, 전자를 개선하면 Slack의 장점까지 흡수할 수 있다.
