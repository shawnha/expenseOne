멀티 회사 지원 — 전체 설계 요약

지금까지 결정된 사항을 바탕으로 한 아키텍처 개요

1. 데이터 모델

DB Schema 변경
+ companies (새 테이블)
id, name, slug,
slack_channel_id, slack_bot_token,
sort_order, is_active,
created_at
~ users (수정)
+ company_id (FK → companies) — 소속 회사
~ expenses (수정)
+ company_id (FK → companies) — 비용이 속한 회사
~ departments (수정)
+ company_id (FK → companies) — 회사별 부서

4. 관리자 대시보드

비용 관리
• 회사 필터 탭 추가
• 전체 / 코리아 / 리테일
• 비용 목록에 회사 라벨 표시
부서 관리
• 부서가 회사에 종속
• 회사별 부서 목록 분리
• 부서 추가 시 회사 선택
회사 관리 (새 페이지)
• 회사 추가/편집/비활성화
• Slack 채널 ID 설정
• 회사별 법인카드 관리 (향후)

변경 범위 요약

영역 변경 내용
DB companies 테이블 신규 + users/expenses/departments에 company_id 추가
제출 폼 법카사용 + 입금요청 폼에 회사 세그먼트 컨트롤 추가
Slack 단일 채널 → 회사별 채널 라우팅 (companies.slack_channel_id)
관리 페이지 회사 관리 페이지 신규 + 기존 페이지에 회사 필터 추가
설정 프로필에 소속 회사 선택 추가
온보딩 최초 가입 시 소속 회사 선택 단계 추가
